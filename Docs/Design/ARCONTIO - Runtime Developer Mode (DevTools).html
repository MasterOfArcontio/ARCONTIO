Runtime Developer Mode (DevTools)

Scopo

Introdurre una modalità sviluppatore attivabile in runtime che permetta di costruire e modificare rapidamente scenari di gioco direttamente nella simulazione.

Questo strumento serve principalmente per:

  • debug del pathfinding
  • test delle meccaniche sociali (furto, fame, sonno, ecc.)
  • costruzione veloce di scenari ripetibili
  • creazione di mappe di test senza ricompilare o modificare codice

La modalità Developer non deve alterare il gameplay normale e deve poter essere attivata/disattivata con un tasto.

Principi architetturali

Separazione View / World

Il sistema DevTools deve rispettare la separazione già esistente in ARCONTIO:

  • World = stato reale della simulazione
  • View/UI = strumenti di editing e debug

La UI non modifica direttamente il World.

Le modifiche devono passare tramite CommandBuffer.

Modifica coerente dello stato

Quando un oggetto viene aggiunto o rimosso devono essere aggiornate tutte le strutture derivate:

  • OcclusionMap
  • LandmarkRegistry
  • walkability cache
  • eventuali altri sistemi derivati

Nella prima versione è accettabile un rebuild globale.

Attivazione della modalità Developer

Toggle modalità

Hotkey principale:

F2 → Toggle Developer Mode

Quando la modalità è attiva:

  • compare un overlay di editing
  • gli input gameplay vengono sospesi o intercettati
  • la UI mostra strumenti e palette oggetti

Interfaccia Developer

Palette oggetti

La UI mostra una palette con oggetti posizionabili.

Esempi iniziali:

  • muro
  • porta
  • cibo
  • letto
  • NPC
  • altri oggetti test

In futuro la palette potrà essere generata automaticamente leggendo object_defs.json.

Tool di editing

Place Tool

Piazza un oggetto sulla cella sotto il mouse.

Click sinistro → piazza oggetto

Erase Tool

Rimuove l’oggetto dalla cella.

Click destro → rimuove oggetto

Brush continuo

Permette di disegnare rapidamente.

Shift + drag → piazza oggetti in continuo

Rectangle Tool (opzionale)

Permette di disegnare muri o oggetti in area.

Click + drag → riempie un rettangolo

Spawn di entità

Spawn oggetti

Gli oggetti vengono creati tramite comando:

PlaceObjectCommand(cell, objectDefId)

Esempio:

PlaceObjectCommand(32,22,"door_wood_good")

Spawn NPC

Permette di creare NPC di test.

SpawnNpcCommand(cell, npcTemplateId)

Esempio:

SpawnNpcCommand(20,20,"citizen_basic")

CommandBuffer

Motivazione

La UI non deve scrivere direttamente nello stato World.

Tutte le operazioni devono passare tramite comandi applicati al tick successivo.

Comandi previsti

PlaceObjectCommand

(cellX, cellY, objectId)

RemoveObjectCommand

(cellX, cellY)

SpawnNpcCommand

(cellX, cellY, npcTemplate)

Aggiornamento delle cache

Dopo ogni modifica è necessario aggiornare:

OcclusionMap

Per mantenere corretta la visione NPC.

LandmarkRegistry

Per mantenere coerente il grafo di navigazione.

Walkability

Per mantenere coerente il pathfinding.

Strategia iniziale (MVP)

Per semplicità:

World.RebuildDerivedCaches()

dopo ogni modifica.

Su mappe piccole è accettabile.

Salvataggio della mappa

La modalità developer deve permettere di salvare la configurazione corrente.

Formato consigliato

JSON semplice.

Esempio:

{
  "width":64,
  "height":64,
  "objects": [
    { "id":"wall_stone", "x":10, "y":10 },
    { "id":"door_wood_good", "x":12, "y":10 }
  ],
  "npcs": [
    { "template":"citizen_basic", "x":20, "y":20, "dir":"E" }
  ]
}

Funzioni previste

SaveMap

SaveDevMap(path)

LoadMap

LoadDevMap(path)

Drag and Drop (opzionale)

Possibile miglioramento dell’interfaccia:

  1. oggetto trascinabile dalla palette
  1. preview ghost sulla griglia
  1. rilascio → spawn oggetto

Non necessario nella prima versione.

Overlay di debug (facoltativo)

Durante Dev Mode può essere utile mostrare informazioni della cella sotto il mouse:

  • coordinate
  • oggetto presente
  • walkable
  • blocks vision
  • landmark presenti
  • NPC presenti

Roadmap di implementazione

DevMode v0 (MVP)

Features:

  • toggle modalità (F2)
  • palette oggetti semplice
  • place tool
  • erase tool
  • rebuild globale cache
  • save/load JSON

Obiettivo:

avere un map editor runtime minimo ma funzionale.

DevMode v1

Miglioramenti:

  • brush continuo
  • rectangle tool
  • spawn NPC
  • orientamento NPC
  • overlay info cella

DevMode v2

Strumenti avanzati:

  • scenari preset
  • invalidazione locale cache
  • palette generata da object_defs.json
  • libreria mappe test

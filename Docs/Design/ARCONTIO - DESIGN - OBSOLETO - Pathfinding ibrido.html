1. Principio guida

💡
Il sistema esistente (GVD-DIN) genera troppi landmark nei corridoi perché lavora su tutta la griglia uniformemente. Il nuovo sistema lavora in due fasi: prima segmenta la mappa in regioni semantiche (stanze vs strozzature), poi genera esattamente un landmark per regione.

Il ragionamento umano nel riconoscere una strozzatura non misura distanze — riconosce prima le regioni, poi la strozzatura emerge automaticamente come confine tra due regioni. Questo sistema replica quel processo.

🔒
Invariante ARCONTIO rispettata: il sistema è un preprocessing statico a map-load. Non dà agli NPC accesso a informazioni che non hanno percepito — produce solo i landmark candidati. L'apprendimento degli NPC rimane soggettivo e via LOS/contatto come da architettura v0.03.

2. Pipeline completa

// Ordine di esecuzione a map-load

1. Distance Transform (DT)          → valore per ogni cella walkable
2. Bridge Detection (Tarjan)         → celle strozzatura marcate
3. Flood Fill sulle regioni          → ogni stanza isolata identificata
4. Landmark per stanza              → DT massimo + mediana ortogonale
5. Landmark per strozzatura         → centroide del gruppo bridge
6. Pruning                          → merge landmark troppo vicini
⚠️
L'intero calcolo avviene una sola volta al caricamento della mappa e produce una lista statica di LandmarkCandidate. Il costo runtime è zero. Gli NPC apprendono questi candidati durante l'esplorazione come già avviene.

3. Passo 1 — Distance Transform

BFS multi-sorgente da tutti i muri simultaneamente. Ogni cella walkable riceve il valore della distanza minima dal muro più vicino.

public static int[,] ComputeDistanceTransform(bool[,] walkable, int width, int height)
{
    var dt = new int[width, height];
    var queue = new Queue<Vector2Int>();

    // Inizializza: muri = 0, celle walkable = -1 (non visitato)
    for (int x = 0; x < width; x++)
    for (int y = 0; y < height; y++)
    {
        if (!walkable[x, y]) {
            dt[x, y] = 0;
            queue.Enqueue(new Vector2Int(x, y));
        } else {
            dt[x, y] = -1;
        }
    }

    // BFS espansione da tutti i muri
    int[] dx = { 0, 0, 1, -1 };
    int[] dy = { 1, -1, 0, 0 };

    while (queue.Count > 0)
    {
        var cell = queue.Dequeue();
        for (int d = 0; d < 4; d++)
        {
            int nx = cell.x + dx[d], ny = cell.y + dy[d];
            if (nx < 0 || nx >= width || ny < 0 || ny >= height) continue;
            if (dt[nx, ny] != -1) continue;
            dt[nx, ny] = dt[cell.x, cell.y] + 1;
            queue.Enqueue(new Vector2Int(nx, ny));
        }
    }
    return dt;
}
📊
Output: griglia int[width, height] dove ogni valore rappresenta la distanza dal muro. Valore 1 = adiacente al muro (corridoio stretto), valore 4+ = centro di stanza ampia.

4. Passo 2 — Bridge Detection con Tarjan

Invece di cercare articulation points su singole celle (che produce falsi positivi nei corridoi lunghi), si costruisce prima un grafo contratto dove le regioni a DT alta sono super-nodi e i corridoi a DT bassa sono archi. Tarjan viene eseguito su questo grafo per trovare i bridge.

4.1 Grafo contratto

// Soglia: celle con DT < soglia sono "corridoio", DT >= soglia sono "stanza"
// La soglia è adattiva: percentile 40 della distribuzione DT della mappa
public static int ComputeAdaptiveThreshold(int[,] dt, int width, int height)
{
    var values = new List<int>();
    for (int x = 0; x < width; x++)
    for (int y = 0; y < height; y++)
        if (dt[x, y] > 0) values.Add(dt[x, y]);

    values.Sort();
    // Percentile 40: separa corridoi (basso DT) da stanze (alto DT)
    return values[(int)(values.Count * 0.4f)];
}

4.2 Bridge detection

// Algoritmo di Tarjan adattato per trovare bridge (archi critici)
// Un bridge è un arco la cui rimozione disconnette il grafo
// → corrisponde a una strozzatura nella mappa

private static int _timer;
private static int[,] _disc, _low;
private static bool[,] _isBridge;

public static bool[,] FindBridgeCells(
    bool[,] walkable, int[,] dt, int width, int height, int threshold)
{
    _timer = 0;
    _disc = new int[width, height];
    _low  = new int[width, height];
    _isBridge = new bool[width, height];

    // Inizializza a -1 (non visitato)
    for (int x = 0; x < width; x++)
    for (int y = 0; y < height; y++)
        _disc[x, y] = -1;

    // Avvia DFS da prima cella walkable a DT bassa (corridoio)
    for (int x = 0; x < width; x++)
    for (int y = 0; y < height; y++)
        if (walkable[x, y] && _disc[x, y] == -1)
            DfsBridge(x, y, -1, -1, walkable, dt, width, height, threshold);

    return _isBridge;
}

private static void DfsBridge(
    int x, int y, int px, int py,
    bool[,] walkable, int[,] dt,
    int width, int height, int threshold)
{
    _disc[x, y] = _low[x, y] = _timer++;
    int[] dx = { 0, 0, 1, -1 };
    int[] dy = { 1, -1, 0, 0 };

    foreach (var (nx, ny) in Neighbors(x, y, dx, dy, width, height))
    {
        if (!walkable[nx, ny]) continue;
        if (nx == px && ny == py) continue; // ignora parent

        if (_disc[nx, ny] == -1)
        {
            DfsBridge(nx, ny, x, y, walkable, dt, width, height, threshold);
            _low[x, y] = Mathf.Min(_low[x, y], _low[nx, ny]);

            // Bridge trovato: arco (x,y)→(nx,ny) è critico
            if (_low[nx, ny] > _disc[x, y])
            {
                // Marca le celle bridge solo se a DT bassa (corridoio)
                if (dt[x, y] < threshold)  _isBridge[x, y]  = true;
                if (dt[nx, ny] < threshold) _isBridge[nx, ny] = true;
            }
        }
        else
        {
            _low[x, y] = Mathf.Min(_low[x, y], _disc[nx, ny]);
        }
    }
}
⚠️
Nota implementativa: la DFS ricorsiva può causare stack overflow su mappe grandi. Convertire in DFS iterativa con stack esplicito per mappe oltre 200x200 celle.

5. Passo 3 — Flood Fill per regioni

Le celle bridge vengono trattate come muri virtuali. Il flood fill isola ogni regione connessa che risulta dalla separazione.

public static int[,] FloodFillRegions(
    bool[,] walkable, bool[,] isBridge, int width, int height)
{
    var regionId = new int[width, height];
    for (int x = 0; x < width; x++)
    for (int y = 0; y < height; y++)
        regionId[x, y] = -1;

    int currentRegion = 0;
    for (int x = 0; x < width; x++)
    for (int y = 0; y < height; y++)
    {
        // Salta muri, bridge già visitati
        if (!walkable[x, y] || isBridge[x, y] || regionId[x, y] != -1) continue;

        // BFS per riempire la regione
        var queue = new Queue<Vector2Int>();
        queue.Enqueue(new Vector2Int(x, y));
        regionId[x, y] = currentRegion;

        while (queue.Count > 0)
        {
            var cell = queue.Dequeue();
            foreach (var n in Neighbors4(cell, width, height))
            {
                if (!walkable[n.x, n.y]) continue;
                if (isBridge[n.x, n.y]) continue; // ← muro virtuale
                if (regionId[n.x, n.y] != -1) continue;
                regionId[n.x, n.y] = currentRegion;
                queue.Enqueue(n);
            }
        }
        currentRegion++;
    }
    return regionId; // ogni cella ha un ID regione, -1 = muro/bridge
}

6. Passo 4 — Landmark per stanza

Per ogni regione identificata dal flood fill vengono applicate due tecniche complementari. I risultati vengono poi uniti.

6.1 Tecnica A — Massimo DT con regola segmento

// Trova il massimo DT per ogni regione
// Se il massimo è un blob quadrato (es. 2x2) → collassa in un punto
// Se il massimo è un segmento lungo → prendi la cella centrale

public static List<Vector2Int> FindDtMaximaForRegion(
    List<Vector2Int> regionCells, int[,] dt)
{
    int maxDt = 0;
    foreach (var c in regionCells)
        maxDt = Mathf.Max(maxDt, dt[c.x, c.y]);

    // Raccoglie tutte le celle con il valore massimo
    var maxCells = regionCells.Where(c => dt[c.x, c.y] == maxDt).ToList();

    // Raggruppa i massimi in segmenti connessi
    var segments = GroupConnected(maxCells);
    var result = new List<Vector2Int>();

    foreach (var segment in segments)
    {
        if (IsBlob(segment)) {
            // Blob quadrato (es 2x2): prendi il centroide
            result.Add(Centroid(segment));
        } else {
            // Segmento lineare: prendi la cella centrale
            result.Add(segment[segment.Count / 2]);
        }
    }
    return result;
}

// Un blob è un gruppo dove larghezza E altezza sono entrambe > 1
private static bool IsBlob(List<Vector2Int> cells)
{
    int minX = cells.Min(c => c.x), maxX = cells.Max(c => c.x);
    int minY = cells.Min(c => c.y), maxY = cells.Max(c => c.y);
    return (maxX - minX) > 0 && (maxY - minY) > 0;
}

6.2 Tecnica B — Mediana ortogonale

// Per ogni cella walkable di una regione:
// lancia raggi nelle 4 direzioni cardinali fino al muro
// la cella è mediana se dist_sinistra ≈ dist_destra (asse H)
// e/o dist_su ≈ dist_giù (asse V)

public static List<Vector2Int> FindMedianCells(
    List<Vector2Int> regionCells,
    bool[,] walkable, int width, int height)
{
    var medians = new List<Vector2Int>();
    var cellSet = new HashSet<Vector2Int>(regionCells);

    foreach (var cell in regionCells)
    {
        int left  = RayToWall(cell, -1,  0, walkable, width, height);
        int right = RayToWall(cell, +1,  0, walkable, width, height);
        int up    = RayToWall(cell,  0, +1, walkable, width, height);
        int down  = RayToWall(cell,  0, -1, walkable, width, height);

        bool isHMedian = Mathf.Abs(left - right) <= 1;
        bool isVMedian = Mathf.Abs(up - down)    <= 1;

        if (isHMedian || isVMedian)
            medians.Add(cell);
    }

    // Raggruppa mediane vicine e prende il centroide di ogni gruppo
    return ClusterAndCentroid(medians, clusterRadius: 2);
}

// Conta celle walkable dal punto fino al muro in una direzione
private static int RayToWall(
    Vector2Int start, int dx, int dy,
    bool[,] walkable, int width, int height)
{
    int dist = 0;
    int x = start.x + dx, y = start.y + dy;
    while (x >= 0 && x < width && y >= 0 && y < height && walkable[x, y])
    {
        dist++;
        x += dx; y += dy;
    }
    return dist;
}
✅
Le due tecniche si coprono a vicenda: la Tecnica A è forte dove la DT ha picchi netti (stanze regolari). La Tecnica B è forte dove la geometria ha simmetria ortogonale (corridoi, stanze irregolari). Casi dove una fallisce, l'altra compensa.

7. Passo 5 — Landmark per strozzatura

// I gruppi di celle bridge connesse diventano ciascuno un landmark
// Il landmark è il centroide del gruppo, scelto tra le celle walkable

public static List<LandmarkCandidate> ExtractChokePointLandmarks(
    bool[,] isBridge, bool[,] walkable, int width, int height)
{
    var visited = new bool[width, height];
    var landmarks = new List<LandmarkCandidate>();

    for (int x = 0; x < width; x++)
    for (int y = 0; y < height; y++)
    {
        if (!isBridge[x, y] || visited[x, y]) continue;

        // BFS per raccogliere il gruppo bridge connesso
        var group = new List<Vector2Int>();
        var queue = new Queue<Vector2Int>();
        queue.Enqueue(new Vector2Int(x, y));
        visited[x, y] = true;

        while (queue.Count > 0)
        {
            var c = queue.Dequeue();
            group.Add(c);
            foreach (var n in Neighbors4(c, width, height))
            {
                if (!isBridge[n.x, n.y] || visited[n.x, n.y]) continue;
                visited[n.x, n.y] = true;
                queue.Enqueue(n);
            }
        }

        // Centroide del gruppo come posizione del landmark strozzatura
        var centroid = NearestWalkable(Centroid(group), walkable, width, height);
        landmarks.Add(new LandmarkCandidate {
            Position = centroid,
            Type = LandmarkType.ChokePoint
        });
    }
    return landmarks;
}

8. Passo 6 — Pruning

// Regole di pruning in ordine di priorità:
//
// 1. BLOB: gruppo di massimi DT a forma quadrata (2x2 o più) → un solo landmark
// 2. VICINANZA: due landmark entro merge_radius celle → tieni quello con DT più alta
// 3. TIPO: non eliminare mai un ChokePoint a favore di un RoomCenter
//          (le strozzature hanno priorità topologica)

public static List<LandmarkCandidate> Prune(
    List<LandmarkCandidate> candidates,
    int[,] dt,
    float mergeRadius = 2.5f)
{
    var result = new List<LandmarkCandidate>(candidates);
    bool changed = true;

    while (changed)
    {
        changed = false;
        for (int i = 0; i < result.Count; i++)
        for (int j = i + 1; j < result.Count; j++)
        {
            float dist = Vector2Int.Distance(result[i].Position, result[j].Position);
            if (dist > mergeRadius) continue;

            // ChokePoint vince sempre su RoomCenter
            if (result[i].Type == LandmarkType.ChokePoint &&
                result[j].Type == LandmarkType.RoomCenter)
            {
                result.RemoveAt(j); changed = true; break;
            }
            if (result[j].Type == LandmarkType.ChokePoint &&
                result[i].Type == LandmarkType.RoomCenter)
            {
                result.RemoveAt(i); changed = true; break;
            }

            // Stesso tipo: tieni quello con DT più alta (più centrale)
            int dtI = dt[result[i].Position.x, result[i].Position.y];
            int dtJ = dt[result[j].Position.x, result[j].Position.y];
            if (dtI >= dtJ) result.RemoveAt(j);
            else           result.RemoveAt(i);
            changed = true; break;
        }
    }
    return result;
}

9. Copertura dei casi critici

Stanza regolare DT massimo Risolto
Stanza a C / irregolare Mediana ortogonale → 2 landmark corretti Risolto
Corridoio stretto lungo Regola segmento centrale sul DT Risolto
Corridoio a L Bridge ai due estremi + mediana nel gomito Risolto

Strozzatura larga (3+ celle) Mediana ortogonale nel buco Risolto

10. Integrazione con sistema esistente

🔌
Il nuovo sistema sostituisce la generazione GVD-DIN ma non tocca il layer di apprendimento degli NPC. NpcLandmarkMemory, LandmarkPathfinder e LandmarkPerceptionSystem rimangono invariati. Cambia solo la sorgente dei candidati.
1
Nuovo file: HybridLandmarkExtractor.cs

Classe statica chiamata da World.cs a map-load. Contiene tutta la pipeline. Produce List<LandmarkCandidate>.

2
Modifica: World.cs

Sostituire la chiamata al generatore GVD-DIN con HybridLandmarkExtractor.Extract(grid). Controllato da flag use_hybrid_extractor in game_params.json per backward compatibility.

3
Modifica: game_params.json

Aggiungere sezione "hybrid_landmark" con i parametri configurabili.

4
Modifica: MapGridLandmarkLabelOverlay.cs

Aggiungere colore distinto per LandmarkType.ChokePoint (es. rosso) vs LandmarkType.RoomCenter (es. azzurro) nel debug overlay.

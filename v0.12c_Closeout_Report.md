COMPLETATA / DONE

---

## Obiettivo checkpoint

Chiudere la dispersione dei parametri runtime tra cartelle e modelli C# non allineati, mantenendo `game_params.json` come file portante della configurazione di ARCONTIO.

Il checkpoint non doveva:

- eliminare `game_params.json`;
- cambiare il formato del file JSON;
- cambiare comportamento simulativo;
- rimuovere `ArcontioLogger`;
- eliminare Telemetry o i canali legacy;
- modificare tick, Save/Load, EL, pannelli o Job/Decision layer.

---

## Esito tecnico

`game_params.json` resta il file centrale e unico da cui parte la configurazione runtime principale.

La patch ha consolidato il lato C# della configurazione:

- `SimulationParams.cs` e `GameParams.cs` sono stati spostati sotto `Assets/Scripts/Core/Config`;
- `SimulationHost` legge `game_params.json` una sola volta come `TextAsset`;
- `SimulationParams` e' diventato il modello principale del bootstrap runtime;
- la configurazione logger ordinaria viene risolta da `SimulationParams`;
- `GameParams.cs` resta come ponte compatibile per test o strumenti isolati;
- il percorso runtime ordinario non dipende piu' da `GameParams.cs`.
